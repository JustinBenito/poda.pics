# Contributing to Podu.pics

Thank you for your interest in contributing to Podu.pics! We love open-source and welcome contributions of all sizes.

## How to Contribute

### 1. Reporting Bugs
- Fill out the bug report template when opening an issue.
- Include steps to reproduce and your environment details.

### 2. Suggesting Features
- Open an issue with the "Feature Request" label.
- Describe the use case and why it would benefit the project.

### 3. Code Contributions
- Fork the repository.
- Create a new branch (`git checkout -b feature/amazing-feature`).
- Ensure your code follows the existing style (run `npm run lint`).
- Commit your changes (`git commit -m 'Add some amazing feature'`).
- Push to the branch (`git push origin feature/amazing-feature`).
- Open a Pull Request.

## Development Workflow

1. **Setup**: Clone your fork and run `npm install`.
2. **Environment**: Configure your `.env.local` (see README.md).
3. **Develop**: Make your changes and test locally using `npm run dev`.
4. **Lint**: Run `npm run lint` before committing.
5. **Build**: Verify the build works with `npm run build`.

## UI & Design Guidelines

- We use **Tailwind CSS** for styling.
- Follow the existing "Glassmorphism" aesthetic.
- Use **Phosphor Icons** for UI elements.
- Keep the interface clean, dark, and focused.

## Code of Conduct

Please be respectful and helpful to others in the community. We aim to foster a welcoming environment for everyone.

Happy coding!
